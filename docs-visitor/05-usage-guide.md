# 사용 가이드

방문자 통계 대시보드 사용 방법입니다.

## 🌐 접속 방법

### 프로덕션 환경
```
https://mp3-free.vercel.app/visitor-stats.html
```

### 로컬 환경
```bash
# 로컬 서버 실행
cd /Users/osam/dev/mp3-free
python3 -m http.server 8000

# 브라우저에서 접속
http://localhost:8000/visitor-stats.html
```

## 📊 대시보드 구성

### 1. 헤더 섹션
```
📊 방문자 통계
페이지 방문 현황 및 분석
                           [얼리버드 관리] 버튼
```

**기능**:
- 페이지 제목 및 설명
- 얼리버드 관리 페이지로 이동

### 2. 통계 카드 (상단)

#### 오늘 방문자
```
오늘 방문자
   42
```
- 오늘 00:00 이후 방문자 수
- 실시간 업데이트

#### 주간 방문자 (7일)
```
주간 방문자 (7일)
     158
```
- 최근 7일간 총 방문자 수
- 일자별 분포 차트 참고

#### 월간 방문자 (30일)
```
월간 방문자 (30일)
     652
```
- 최근 30일간 총 방문자 수
- 장기 트렌드 파악

### 3. 차트 섹션

#### 일자별 방문자 차트
```
일자별 방문자 (최근 7일)
[선형 차트]
```

**표시 정보**:
- X축: 날짜 (MM/DD 형식)
- Y축: 방문자 수
- 데이터: 최근 7일간 일별 방문자 수
- 스타일: 파란색 선, 면적 채우기

**사용법**:
- 차트 위에 마우스 오버: 정확한 수치 확인
- 트렌드 파악: 증가/감소 추세 확인

#### 디바이스별 방문자 차트
```
디바이스별 방문자
[도넛 차트]
```

**표시 정보**:
- Desktop: 파란색
- Mobile: 초록색
- Tablet: 노란색

**사용법**:
- 각 섹션 클릭: 해당 디바이스 하이라이트
- 범례 클릭: 특정 디바이스 숨기기/표시

#### 브라우저별 방문자 차트
```
브라우저별 방문자
[막대 차트]
```

**표시 정보**:
- Chrome, Safari, Firefox, Edge 등
- 막대 높이: 방문자 수

**사용법**:
- 막대 위에 마우스 오버: 정확한 수치
- 브라우저 점유율 파악

### 4. 최근 방문자 테이블

#### 헤더
```
최근 방문자 (20명)          [📥 CSV 내보내기] 버튼
```

#### 테이블 컬럼
| 방문일시 | 페이지 | 디바이스 | 브라우저 | 유입경로 | 해상도 |
|---------|--------|---------|---------|---------|--------|
| 2025-11-14 15:30 | / | Desktop | Chrome | https://google.com | 1440x900 |

**컬럼 설명**:
- **방문일시**: 방문 시각 (YYYY-MM-DD HH:MM)
- **페이지**: 방문한 페이지 경로
- **디바이스**: Desktop, Mobile, Tablet
- **브라우저**: Chrome, Safari, Firefox 등
- **유입경로**: Referrer URL 또는 "Direct"
- **해상도**: 뷰포트 크기

## 🔄 실시간 업데이트

### 자동 새로고침
- 페이지 로드 시 최신 데이터 가져오기
- Firestore 실시간 리스너로 변경 사항 자동 반영
- 새 방문자 발생 시 테이블 자동 업데이트

### 수동 새로고침
브라우저 새로고침 (F5 또는 Cmd+R)로 전체 데이터 갱신

## 📥 CSV 내보내기

### 1. 버튼 위치
"최근 방문자" 테이블 상단 우측

### 2. 사용 방법
1. "📥 CSV 내보내기" 버튼 클릭
2. 브라우저 다운로드 폴더에 파일 저장
3. 파일명: `visitors_[timestamp].csv`

### 3. CSV 파일 형식
```csv
방문일시,페이지,Referrer,디바이스,브라우저,해상도,User Agent
2025-11-14 15:30,/,direct,Desktop,Chrome,1440x900,"Mozilla/5.0..."
2025-11-14 15:25,/,https://google.com,Mobile,Safari,375x812,"Mozilla/5.0..."
```

### 4. Excel에서 열기
1. CSV 파일 더블클릭 (Excel 자동 실행)
2. UTF-8 인코딩으로 저장됨 (한글 정상 표시)

### 5. 활용 방법
- 데이터 분석 (피벗 테이블, 차트)
- 외부 시스템으로 데이터 이전
- 백업 및 아카이빙

## 🔍 데이터 분석 팁

### 1. 일자별 트렌드 분석
**목적**: 방문자 증가/감소 추세 파악

**방법**:
1. 일자별 차트 확인
2. 특정 날짜의 급증/급감 원인 분석
3. 요일별 패턴 파악 (주말 vs 평일)

### 2. 디바이스별 최적화
**목적**: 주요 디바이스에 맞춘 UX 최적화

**방법**:
1. 디바이스별 차트 확인
2. 주요 디바이스 파악 (예: Mobile 70%)
3. 해당 디바이스에 맞춘 반응형 디자인 강화

### 3. 브라우저 호환성
**목적**: 주요 브라우저 지원 확인

**방법**:
1. 브라우저별 차트 확인
2. 소수 브라우저 사용자 파악
3. 해당 브라우저 테스트 및 버그 수정

### 4. 유입 경로 분석
**목적**: 마케팅 효과 측정

**방법**:
1. 최근 방문자 테이블에서 Referrer 컬럼 확인
2. CSV 내보내기 → Excel 피벗 테이블 생성
3. 주요 유입 경로 파악 (검색엔진, 소셜미디어 등)
4. 효과적인 채널 집중

### 5. 시간대별 방문 패턴
**목적**: 최적 콘텐츠 발행 시간 결정

**방법**:
1. CSV 내보내기
2. Excel에서 시간대별 그룹화
3. 피크 타임 파악
4. 해당 시간에 마케팅 활동 집중

## 🎯 주요 지표 해석

### 방문자 수 (DAU, WAU, MAU)
- **DAU (Daily Active Users)**: 오늘 방문자
- **WAU (Weekly Active Users)**: 주간 방문자
- **MAU (Monthly Active Users)**: 월간 방문자

**정상 범위**:
- 소규모 사이트: DAU 10-100, WAU 50-500, MAU 200-2,000
- 중규모 사이트: DAU 100-1,000, WAU 500-5,000, MAU 2,000-20,000

### 디바이스 분포
**일반적인 비율**:
- Desktop: 40-60%
- Mobile: 35-55%
- Tablet: 5-10%

**해석**:
- Mobile > Desktop: 모바일 최적화 중요
- Desktop > Mobile: 업무/학습 관련 콘텐츠

### 브라우저 분포
**일반적인 비율**:
- Chrome: 60-70%
- Safari: 15-25%
- Firefox: 3-8%
- Edge: 3-8%

**해석**:
- 주요 브라우저 우선 지원
- 소수 브라우저 호환성 테스트

### Referrer (유입 경로)
**주요 카테고리**:
- **Direct**: 30-50% (즐겨찾기, URL 직접 입력)
- **Search Engine**: 20-40% (Google, Naver 등)
- **Social Media**: 10-30% (Facebook, Instagram 등)
- **기타**: 5-10% (이메일, 광고 등)
